# ADB Pull Photos 图形界面 PRD

## 1. 产品概述

ADB Pull Photos 是一个面向 Mac 用户的 Android 照片导出工具。当前版本通过命令行和配置文件从手机目录批量拉取照片，并使用 SQLite 记录传输历史，避免重复导出。下一阶段需要增加图形交互界面，让用户可以直观看到手机连接状态、相册目录、照片缩略图，并选择指定照片传输到 Finder 中的目标目录。

## 2. 背景与问题

当前脚本已经支持：

- 通过 `adb devices` 检测已授权 Android 设备。
- 从配置文件读取多个相册任务，例如 Camera、Screenshots、Facebook、WeiXin。
- 从手机目录递归枚举文件。
- 使用传输日志跳过已导出照片。
- 支持严格模式、快速模式、强制重传、补记本地已有文件等选项。

现有问题：

- 用户必须编辑 `cfg/pull_camera.ini` 才能管理相册目录。
- 无法在传输前预览照片内容。
- 无法选择部分照片，只能按目录批量执行。
- 传输目标目录固定在配置里，不符合临时导出场景。
- 手机连接、授权、传输进度和失败原因都隐藏在终端输出中。

## 3. 目标

### 3.1 用户目标

- 连接 Android 手机后，立即知道是否连接成功。
- 看到手机中的可导出相册目录和相册名称。
- 点开相册后看到照片缩略图。
- 支持单选、多选、全选和取消选择。
- 点击传输后，通过 Finder 选择保存位置。
- 传输过程中看到进度、成功、跳过和失败状态。

### 3.2 产品目标

- 将命令行导出流程产品化为一个易用的 Mac 桌面界面。
- 保留现有 adb 和 SQLite 去重能力。
- 降低误操作风险，尤其是重复导出、传错目录、连接失败但用户无感等问题。
- 为后续增加更多来源目录、搜索、过滤和断点续传打基础。

## 4. 范围

### 4.1 MVP 范围

- 手机连接状态检测。
- 相册目录列表。
- 相册内照片和视频缩略图网格。
- 照片选择能力：单选、全选、Shift 连续多选、Command 离散多选。
- Finder 目标目录选择。
- 传输进度与完成结果。
- 错误提示与重试入口。

### 4.2 暂不包含

- iOS 设备导入。
- 云相册同步。
- 手机端删除照片。
- 照片编辑、裁剪、压缩。
- 视频转码。
- 跨相册同时选择和传输。
- 多台手机同时导入的高级管理。

## 5. 用户画像

主要用户是使用 Mac 管理 Android 手机照片的人，具备基本 Finder 使用经验，但不希望频繁使用终端。用户可能需要定期导出相机、截图、微信、社交 App 图片到 Mac 备份或整理。

## 6. 核心使用流程

1. 用户打开应用。
2. 应用自动检查 adb 是否可用。
3. 应用检测是否有已授权 Android 设备连接。
4. 如果连接成功，顶部显示设备名称或序列号，以及“已连接”状态。
5. 应用读取配置中的相册目录，并展示相册列表。
6. 用户点击一个相册。
7. 应用扫描该相册下的照片文件，并生成缩略图网格。
8. 用户选择照片，可单选、多选或全选。
9. 用户点击“传输”。
10. 系统打开 Finder 目录选择窗口。
11. 用户选择本地保存目录。
12. 应用开始传输选中的照片。
13. 应用展示传输进度、当前文件、成功数、跳过数、失败数。
14. 传输完成后，用户可打开 Finder 查看结果。

## 7. 信息架构

### 7.1 主界面布局

主界面分为四个区域：

- 顶部状态栏：设备连接状态、刷新按钮、设置入口。
- 左侧相册栏：相册名称、远端路径、照片数量、扫描状态。
- 中央照片区：缩略图网格、选择状态、文件基础信息。
- 底部操作栏：已选数量、全选、取消选择、传输按钮、传输进度。

### 7.2 相册列表字段

每个相册项显示：

- 相册名称，例如 Camera、Screenshots、WeiXin。
- 远端目录路径，例如 `/sdcard/DCIM/Camera`。
- 照片数量。
- 已导出数量，来自 SQLite 日志。
- 扫描状态：未扫描、扫描中、已扫描、扫描失败。

### 7.3 媒体缩略图字段

每个媒体文件显示：

- 缩略图。
- 文件名。
- 选择勾选状态。
- 已导出标记。
- 文件大小。
- 拍摄或修改时间。
- 媒体类型：图片或视频。

## 8. 功能需求

### 8.1 手机连接状态

应用启动后自动检测 adb：

- 如果找不到 adb，显示“未安装 adb”，并提供安装说明入口。
- 如果 adb 存在但没有设备，显示“未连接手机”。
- 如果设备未授权，显示“等待手机授权”，提示用户在手机上允许 USB 调试。
- 如果检测到已授权设备，显示“已连接”。
- 如果配置了固定 `device_id` 但当前未检测到该设备，显示“指定设备未连接”。

连接状态需要支持手动刷新。

### 8.2 相册目录选择

相册来源优先读取 `cfg/pull_camera.ini` 中启用的 `job.*` 配置：

- `job.camera` 显示为 Camera。
- `job.screenshots` 显示为 Screenshots。
- `job.facebook` 显示为 Facebook。
- `job.weixin` 显示为 WeiXin。

用户点击相册后进入该相册的照片网格。MVP 可先不支持在 UI 内新增相册目录，但需要预留“添加相册”入口。

后续版本支持：

- 用户输入手机端目录路径。
- 设置相册显示名称。
- 保存到配置文件。
- 启用或停用某个相册。

### 8.3 媒体扫描与缩略图

点击相册时，应用调用 adb 扫描远端目录中的媒体文件。

支持的文件类型：

- 图片：`.jpg`、`.jpeg`、`.png`、`.webp`、`.heic`、`.gif`
- 视频：`.mp4`、`.mov`、`.mkv`、`.3gp`、`.webm`

缩略图生成策略：

- MVP 推荐将远端媒体临时拉取到本机缓存目录后生成缩略图。
- 图片缩略图使用 macOS 原生图像解码能力生成。
- 视频缩略图优先使用 AVFoundation 从视频首帧或 1 秒位置生成。
- 缩略图应缓存在本地临时目录，避免重复加载。
- 缩略图加载失败时显示文件占位图和文件名。
- 大相册需要分页或虚拟滚动，避免一次渲染过多图片造成卡顿。

### 8.4 照片选择

选择规则：

- 单击照片：选中该照片，并取消其他选择。
- Command + 单击：切换当前照片选中状态，不影响其他照片。
- Shift + 单击：从上一次选中的照片到当前照片之间连续选中。
- 全选：选择当前相册当前过滤范围内所有照片。
- 取消选择：清空当前选择。
- 切换相册时清空当前选择；MVP 不支持跨相册选择和传输。

选择反馈：

- 被选照片显示明显描边和勾选标记。
- 底部显示“已选择 N 张”。
- 未选择照片时，“传输”按钮不可用。

### 8.5 传输目标目录选择

用户点击“传输”后，系统打开 Finder 目录选择窗口。

要求：

- 只能选择文件夹，不能选择文件。
- 默认打开上一次成功使用的保存目录。
- 用户取消选择时，不开始传输，保留当前照片选择。
- 选择目录后，应用将选中照片保存到该目录。

保存路径策略：

- 默认直接保存到用户选择的文件夹。
- 如果用户从多个相册选择照片，建议按相册名称建立子目录。
- 文件名冲突时默认跳过已存在且大小一致的文件。
- 如果文件名相同但大小不同，自动追加序号，例如 `IMG_0012 (1).jpg`。

### 8.6 传输

传输使用现有 `adb pull` 能力。

传输前：

- 显示目标目录。
- 显示待传输文件数和预计总大小。
- 可选显示“跳过已导出文件”开关。

传输中：

- 显示整体进度。
- 显示当前正在传输的文件名。
- 显示成功、跳过、失败数量。
- 禁用会破坏状态的操作，例如切换相册或修改目标目录。
- 允许取消传输，取消后保留已成功传输文件和日志。

传输后：

- 显示完成摘要。
- 提供“打开 Finder”按钮。
- 失败文件可展开查看，并支持重试失败项。

### 8.7 去重与传输历史

沿用现有 SQLite 逻辑：

- 严格模式：按 `remote_path + remote_size + remote_mtime` 判断是否传过。
- 快速模式：按文件名判断是否传过。
- 本地已有且大小一致时可跳过。
- 成功传输后写入日志。

UI 中需要将已导出的照片标记为“已导出”。用户仍可通过“强制重新传输”选项重新导出。

## 9. 状态与异常

### 9.1 空状态

- 未连接手机：显示连接引导。
- 已连接但无相册：显示“没有配置相册目录”。
- 相册为空：显示“此相册没有找到照片”。
- 无选择：底部提示“选择照片后开始传输”。

### 9.2 错误状态

- adb 不存在。
- 手机未授权。
- 指定设备未连接。
- 相册目录不存在或无权限。
- 缩略图加载失败。
- 目标目录无写入权限。
- 传输过程中 USB 断开。
- 单个文件传输失败。

所有错误提示需要包含：

- 发生了什么。
- 用户可以怎么处理。
- 是否可以重试。

## 10. 交互细节

### 10.1 顶部状态栏

状态文案：

- 未检测到 adb
- 未连接手机
- 等待授权
- 已连接：{device_id}
- 指定设备未连接

操作：

- 刷新连接
- 设置

### 10.2 左侧相册栏

相册项点击后选中。当前相册需要保持高亮。相册扫描中显示加载状态。

### 10.3 照片网格

网格需要支持：

- 自适应窗口宽度。
- 缩略图固定比例。
- 滚动加载。
- 键盘辅助选择。

建议快捷键：

- `Command + A`：全选。
- `Escape`：取消选择。
- `Space`：预览当前照片，后续版本可做。

### 10.4 底部操作栏

内容：

- 已选择数量。
- 全选按钮。
- 取消选择按钮。
- 传输按钮。
- 传输进度。

按钮规则：

- 未选择照片时，传输按钮禁用。
- 未连接设备时，传输按钮禁用。
- 传输中，传输按钮变为“传输中”或显示取消入口。

## 11. 非功能需求

### 11.1 性能

- 1000 张照片的相册应在可接受时间内完成列表扫描。
- 缩略图应渐进加载，不阻塞主界面。
- UI 滚动需要保持流畅。

### 11.2 可靠性

- 传输中断不会破坏已完成文件。
- 使用 `.part` 临时文件，成功后再替换为最终文件。
- 文件大小校验失败时保留失败状态，不写入成功日志。

### 11.3 隐私

- 不上传照片到任何远端服务。
- 缩略图缓存仅保存在本机。
- 用户可清理缩略图缓存。

### 11.4 可访问性

- 连接状态不能只依赖颜色表达。
- 选择状态需要有勾选图标或明确标识。
- 关键操作支持键盘。

## 12. 技术建议

### 12.0 技术栈

MVP 使用原生 Swift 开发 macOS 桌面应用。

推荐技术组合：

- SwiftUI：构建主界面、相册侧栏、媒体网格和底部操作栏。
- AppKit：处理 Finder 目录选择窗口，例如 `NSOpenPanel`。
- Foundation `Process`：调用 `adb devices`、`adb shell`、`adb pull`。
- SQLite.swift 或系统 SQLite C API：读写传输记录数据库。
- AVFoundation：为常见视频格式生成缩略图。
- QuickLookThumbnailing：作为图片或视频缩略图的补充方案。

### 12.1 架构

建议将现有脚本拆为三层：

- ADB 服务层：检测设备、扫描目录、读取文件信息、执行 pull。
- 传输记录层：封装 SQLite 查询和写入。
- UI 层：相册列表、缩略图、选择、传输进度。

### 12.2 可复用现有函数

可复用：

- `check_adb`
- `adb_cmd`
- `adb_shell_text`
- `list_remote_files`
- `get_remote_stat`
- `connect_db`
- `has_successful_transfer`
- `has_transferred_filename`
- `pull_file`
- `record_success`
- `record_fast_success`
- `load_app_config`

需要改造：

- `check_adb` 当前会 `sys.exit(1)`，UI 版本应返回结构化状态。
- `run_job` 当前按整个目录执行，UI 版本需要支持传入选中文件列表。
- 当前日志输出为 `print`，UI 版本需要事件回调或进度对象。

### 12.3 推荐 Swift API 形态

```swift
struct DeviceStatus {
    let adbAvailable: Bool
    let connected: Bool
    let authorized: Bool
    let deviceID: String?
    let message: String
}

struct Album {
    let id: String
    let name: String
    let remotePath: String
    let enabled: Bool
}

struct RemoteMedia: Identifiable {
    let id: String
    let remotePath: String
    let filename: String
    let mediaType: MediaType
    let size: Int64?
    let modifiedAt: Date?
    let albumID: String
    let transferred: Bool
}

enum MediaType {
    case image
    case video
}

protocol ADBService {
    func deviceStatus(configDeviceID: String?) async -> DeviceStatus
    func listMedia(in album: Album, deviceID: String?) async throws -> [RemoteMedia]
    func pull(_ media: RemoteMedia, to targetDirectory: URL) async throws
}
```

## 13. 验收标准

### 13.1 连接状态

- 断开手机时，界面显示未连接。
- 手机未授权时，界面显示等待授权。
- 手机授权成功后，界面自动或手动刷新为已连接。
- 配置了固定设备但连接了其他设备时，界面能明确提示。

### 13.2 相册和缩略图

- 能读取配置文件里的 Camera、Screenshots、Facebook、WeiXin。
- 点击相册后显示该相册下的照片和视频缩略图。
- 常见视频格式能显示缩略图或明确的视频占位图。
- 缩略图加载失败不会导致应用崩溃。
- 大量照片滚动时界面不卡死。

### 13.3 选择

- 单击可选中单张照片。
- `Command + 单击` 可离散多选。
- `Shift + 单击` 可连续多选。
- 全选可以选中当前相册全部照片。
- 取消选择可以清空选择。
- 切换相册会清空当前选择。
- 不能跨相册同时选择和传输。
- 已选择数量显示正确。

### 13.4 传输

- 未选择照片时不能点击传输。
- 点击传输后弹出 Finder 目录选择窗口。
- 取消 Finder 选择不会开始传输。
- 选择目录后只传输用户选中的照片。
- 传输完成后可打开 Finder 查看文件。
- 已成功传输的文件会写入 SQLite 日志。
- 失败文件有明确提示并可重试。
